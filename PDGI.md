# How IndiaTemperatureMap follows PDGI

An open, hyperlocal India temperature and heat platform (Leaflet, multi-source). Built on **People's Digital Goods & Infrastructure (PDGI)** principles — people before digital, rights-centric, commons-oriented, transparent ([pdgi.org](https://pdgi.org/)).

This is a public scorecard of how those principles are implemented here, with honest notes on where it falls short. It was generated from a repository audit (2026-07); update it as the code changes.

Status key: ✅ implemented · 🟡 partial · ⛳ gap · — not applicable.

## Scorecard

### People before digital — ✅
Faithful multi-model rendering; per-source NOTICE attributions; spread flags.

### Transparency and accountability — ✅
Open repo; Python fetchers; public sources; methodology in the README.

### Decentralisation and no lock-in — 🟡
No account, forkable; JSON/CSV data files but no in-app export button.

### Free software and the digital commons — 🟡
Open code; the non-commercial clause is being dropped (BY-NC-SA → BY-SA) to reach the commons.

### Privacy — ✅
No third-party analytics or trackers; no PII collected.

### Platform cooperativism — 🟡
Correction path added (CONTRIBUTING + issue template); no in-app contribution.

### Humans in the loop (AI does not cut people out) — —
No AI / MCP / API surface.

### A non-digital alternative — 🟡
CSV/JSON artifacts in the repo; the app is JS-only with no no-JS / print view.

### Grassroots and reaching the divide-affected — ⛳
English-only, expert-oriented UI. Direction: vernacular access.

### Algorithmic fairness — —
A visualisation/catalog; it makes no decisions about people.

**Overall:** 3✅ 5🟡 1⛳
